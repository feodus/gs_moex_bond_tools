# MOEX Bond Data for Google Sheets - Product Description

## Назначение проекта

Проект представляет собой библиотеку Google Apps Script для получения данных по облигациям с Московской Биржи (MOEX) в реальном времени прямо в Google Таблицы. Это решение позволяет пользователям получать актуальные данные об облигациях (цены, даты купонов, размеры купонов, даты погашения и т.д.) без необходимости самостоятельного обращения к API MOEX.

## Решаемые проблемы

1. **Сложность доступа к данным об облигациях MOEX** - API MOEX требует знаний по работе с HTTP-запросами, форматированию данных и т.д.
2. **Необходимость обработки различных форматов данных** - MOEX предоставляет данные в различных форматах, которые нужно корректно интерпретировать.
3. **Ограничения API** - необходимо управлять частотой запросов к API, чтобы избежать блокировок.
4. **Необходимость кэширования данных** - для оптимизации производительности и соблюдения лимитов API.
5. **Сложность работы с флоатерами** - облигации с плавающей ставкой требуют специальной обработки для получения размера следующего купона.

## Как должно работать решение

1. Пользователь устанавливает скрипт в Google Таблицы, следуя инструкции.
2. В таблице становится доступно меню "MOEX" с возможностью обновления всех данных.
3. Пользователь может использовать кастомные функции в ячейках таблицы, такие как:
   - `=GET_MOEX_PRICE("TICKER")` - для получения текущей цены облигации
   - `=GET_MOEX_NAME("TICKER")` - для получения наименования облигации
   - `=GET_NEXT_COUPON("TICKER")` - для получения даты следующего купона
   - `=GET_COUPON_VALUE("TICKER")` - для получения размера следующего купона
   - `=GET_MATURITY_DATE("TICKER")` - для получения даты погашения
   - `=GET_NEAREST_OPTION_DATE("TICKER")` - для получения ближайшей даты опциона (put/call) или амортизации
4. Система автоматически кэширует полученные данные с установленными интервалами.
5. При необходимости пользователь может принудительно обновить все данные через меню MOEX.

1. Упростить доступ к данным об облигациях MOEX
2. Обеспечить надежное и стабильное получение данных
3. Обеспечить масштабируемость и возможность работы с большим количеством облигаций

5. Обеспечить поддержку различных типов облигаций, включая флоатеры
